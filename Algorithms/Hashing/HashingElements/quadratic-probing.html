Quadratic Probing: In-Depth Explanation

What is Quadratic Probing?

Quadratic probing is a collision resolution technique that uses quadratic polynomial probing. When a collision occurs, instead of checking consecutive slots, it checks slots at quadratic intervals.

Key Characteristics:

  • Probe Sequence: When collision occurs at position h(k), try positions:
    • h(k) + 1² mod m
    • h(k) + 2² mod m
    • h(k) + 3² mod m
    • And so on...
  • Formula: h'(k) = (h(k) + i²) mod m
    • where i is the probe number (1, 2, 3, ...)
    • m is the table size

Comparison with Linear Probing

  • Better clustering behavior: Quadratic probing helps avoid the primary clustering that occurs with linear probing
  • Jump sequence: Instead of checking consecutive locations, it jumps quadratically (1, 4, 9, 16, ...) slots away
  • Secondary clustering: Keys that hash to the same initial position will follow the same probe sequence

Advantages and Disadvantages

Advantages:

  • Reduces primary clustering compared to linear probing
  • Better performance than linear probing for medium-load factors
  • Simple to implement

Disadvantages:

  • May not probe all table positions when table size is not prime
  • Can suffer from secondary clustering
  • Less cache-friendly than linear probing due to non-sequential probing
